Look for subscription attempts that were made before the publisher was created and claim any matches.

When the subscriber runs before the publisher, the subscription process cannot complete so we store the parameters for that attempt off as an orphan to be adopted later.

Transfer data that has been scheduled through the register_transfer() function to be transfered in this period.

This function is at the heart of multi-rate publish/subscribe. This is where the data gets moved between the seperate buffers allocated to each rategroup.
